Introduction EMT H2H API

Overview of Integration API
Base URL
  • https://api.ptx.xyz (Production Server Live Data)
  • https://api-sbx.ptx.xyz (Sandbox Server Test Data)
Request
Method Endpoint Usage
POST /auth/token Get Access Token (auth_token) for Oauth2
POST /auth/refresh Get a new auth_token
POST /topup Send topup request
GET /topup/{slug} Get Topup Detail By Slug
GET /topup/?ref_code={ref_code} Get Detail of Specific Topup by using its reference Code when using ref_code if more than one Record is found only the latest one will be returned
GET /balance Get wallet data
GET /history/topup Topup request history data
GET /history/wallet Successful transaction data history/entered balance history

Authentication Oauth2 (Access Token)

Access Token is used to identify a user or device.
Request
Setting Value
HTTP Method POST
Path /auth/token
Host https://api-sbx.ptx.xyz
Request Headers
Name Format Mandatory Description
Authorization Basic base64(client_id:client_secret) Yes
Content-Type application/json Yes
Request
Name Format Mandatory Description
grant_type String Yes Value = client_credentials
Result of the request will contain the following information :
Response
Field DataType Description
access_token String Your access_token
token_type String Default is Bearer
expires_in String access_token validity, in x-seconds
refresh_token String Used instead of client_id:client_secret but is one time use to get new auth_token
scope String Application scope / permission granted to application

TOPUP

Topup is used to send topup request.
Request
Setting Value
HTTP Method POST
Path /topup
Host https://api-sbx.ptx.xyz
Request Headers
Name Format Mandatory Description
Authorization Bearer access_token Yes
Content-Type application/json Yes
Request Body
Name Format Mandatory Description
customer_name String Yes Customer Name
customer_email String Customer Email
nominal oneOf( String or Number) Yes Topup Nominal
ref_code String
maxLength: 128		 Reference Code for Merchant Tracking
note String Transaction Note
mode String
Enum: va, qris, null		 Topup Mode. default to 'va' if `null` or not defined
callback_url String Custom Callback URL. Default to What defined in API
callback_type String Callback Data Format. Default to `form` if not defined
Result of the request will contain the following information :
Response
Field DataType Description
slug String Unique Transaction Identity
nominal String Topup Nominal
ref_code String Reference Code
customer_name String
customer_email String
mode String
status String Status of The Topup :
  • PENDING - Topup Request is Created and Waiting for Payment
  • PAID - Topup Request is Paid and Waiting for Processing
  • FINISH - Topup Request is Processed and Wallet Balance Updated
Invoice_data Object The Payment that need to be done
Properties from Invoice_data :
bank String Bank Name
account String Bank Account Number
name String Account Owner Name
howto String Link to How to Pay Tutorial
amount String Amount that has to be paid

Example

POST data
POST https://api-sbx.ptx.xyz/topup
Content-Type: application/json
Authorization : Bearer acs_Goi9ms7p3qwjccMVAuWEMOw-YV04l_HM 

                       {
                        "customer_name": "Ms. Dave Cummings",
                        "nominal": 1000000
                        }

                      {
                      "slug": "XU3zaor88bFE",
                      "nominal": "1000000.00",
                      "ref_code": null,
                      "customer_name": "Ms. Dave Cummings",
